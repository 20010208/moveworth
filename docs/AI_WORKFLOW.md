# AI開発・作業記録・引き継ぎ運用仕様書

最終更新: 2026-07-21
対象: Claude Code / Codex / Sonnet / ChatGPT

## 1. 目的

本仕様は、Claude CodeまたはCodexの利用上限が突然到達しても、別のエージェントがチャット履歴に依存せず作業を継続できる状態を維持するためのもの。

実現すること：

- 作業開始時から引き継ぎ情報を残す
- 現在地と履歴を分離する
- Gitとメモの役割を分離する
- ファイル肥大化を防ぐ
- 未解決事項と恒久判断を消失させない
- Claude CodeとCodexが同じ安全ルールに従う

## 2. 役割

### ユーザー

CEO・プロダクトオーナー。優先順位、仕様、公開、危険操作を最終承認する。

### Sonnet

PM・テックリード。問題分析、指示作成、Output評価、次の指示、設計判断の整理を行う。

### Claude Code

第一実装担当。通常のメイン実装、複雑な既存文脈を要する変更、調査、テストを担当する。

### Codex

第二実装担当・フォールバック実装者。Claude Code上限時の継続、独立タスク、別調査、テストを担当する。

## 3. 正とする保存先

| 情報 | 保存先 |
|---|---|
| コードの具体的変更 | Git diff / commit |
| 現在進行中の状態 | `.ai/CURRENT_HANDOFF.md` |
| 直近3日間の作業履歴 | `.ai/RECENT_ACTIVITY.md` |
| 恒久的な設計・運用判断 | `.ai/DECISIONS.md` |
| 将来の未完了タスク | `docs/BACKLOG.md` |
| Claude固有入口 | `CLAUDE.md` |
| Codex固有入口 | `AGENTS.md` |

メモへコード全文、diff全文、長いログを複製しない。

## 4. 作業開始手順

エージェントはコード変更前に以下を行う。

1. 自分の入口ファイルを読む
2. `CLAUDE.md` のMoveWorth安全ルールを読む
3. `.ai/DECISIONS.md` を読む
4. `.ai/CURRENT_HANDOFF.md` を読む
5. `git status` を実行
6. `git diff` を確認
7. 未コミット変更の目的を把握
8. `.ai/CURRENT_HANDOFF.md` を今回の状態へ更新

### 開始時に記録する内容

- タスクID
- 担当エージェント
- 目的
- 現在確認できる問題
- 実装・調査予定
- 変更する可能性がある主要ファイル
- 既存の未コミット変更
- 禁止事項・危険操作
- ユーザー判断が必要になりそうな点

## 5. 作業中の更新

終了時だけではなく、以下のタイミングで `CURRENT_HANDOFF.md` を更新する。

- 意味のある実装単位が完了したとき
- 当初の方針を変更したとき
- 新しい問題を発見したとき
- 仮説が誤りと判明したとき
- lint / typecheck / test / buildを実行したとき
- DB操作を実行したとき
- 危険操作・一括処理の直前
- ユーザー判断が必要になったとき

### 更新頻度の目安

- 小タスク: 開始時 + 完了時
- 中タスク: 開始時 + 実装完了 + 検証完了
- 長時間タスク: 30〜60分または主要な区切りごと

細かな1ファイル編集ごとには更新せず、意味のある状態変化で更新する。

## 6. `CURRENT_HANDOFF.md` 仕様

- 現在のタスク1件だけを保持する
- 履歴を追記し続けない
- タスクが変わったら上書きする
- 原則200行以内、目安12KB以内
- 次の担当がチャットを読まずに再開できる内容にする

必須項目：

```md
# Current Handoff

最終更新:
最終担当:
タスクID:
状態: 待機 / 調査中 / 実装中 / 検証中 / 中断 / 完了

## 目的
## 現在の状態
## 完了した作業
## 変更した主要ファイル
## Git状態・未コミット変更
## 実行済みの検証
## 未実行の検証
## 未解決事項
## 次に行う作業
## 禁止事項・注意事項
## ユーザー判断が必要な事項
```

## 7. `RECENT_ACTIVITY.md` 仕様

- Claude CodeとCodexの直近3日分を時系列で保持する
- 1実行あたり原則30行以内
- 同じ内容を繰り返さない
- コード・diff・ログ全文を貼らない

記録項目：

- 日時・タイムゾーン
- 担当
- タスクID
- 状態
- 実施内容
- 主要な変更ファイル
- 検証結果と対象件数
- 未解決事項
- 次の担当・次の作業
- 関連コミット

## 8. 3日経過後の整理

古い記録を即座に消さず、次を確認してから整理する。

### 削除・圧縮できるもの

- 完了済み作業の細かな経過
- 解消済みの一時的原因候補
- 成功コマンドの詳細
- Gitで確認可能な変更一覧
- 重複説明

### 移動してから削除するもの

- 未完了・将来タスク → `docs/BACKLOG.md`
- 恒久判断・禁止事項・再発防止策 → `.ai/DECISIONS.md`
- 重大事故や長期参照価値のある経緯 → `.ai/archive/YYYY-MM.md`

未解決事項、承認待ち、再発防止策を3日で消してはならない。

## 9. `DECISIONS.md` 仕様

恒久判断をADR風に記録する。

```md
## DEC-YYYYMMDD-NN: タイトル

- 日付:
- 状態: 採用 / 暫定 / 廃止
- 決定者:

### 決定
### 理由
### 影響範囲
### 代替案・却下理由
### 見直し条件
```

廃止した判断も削除せず、状態を「廃止」にして新判断への参照を残す。

## 10. `BACKLOG.md` 仕様

未実施タスクは次の形式で管理する。

```md
## BL-YYYYMMDD-NN: タイトル

- 優先度: 高 / 中 / 低
- 状態: 未着手 / 調査中 / 保留 / 完了
- 登録日:
- 関連領域:
- 前提・ブロッカー:
- 完了条件:
- 関連ファイル・コミット:
```

チャット上の合意だけで残さない。

## 11. エージェント交代

### Claude Code → Codex

1. `CURRENT_HANDOFF.md`
2. `RECENT_ACTIVITY.md`
3. `git status`
4. `git diff`
5. 必要に応じて直近コミット
6. 未実行検証

を確認してから継続する。

### Codex → Claude Code

同じ手順を適用する。前担当の変更を無条件に書き直さない。

### 突然のトークン切れ

終了時の引き継ぎ作成に依存しない。最後に更新された `CURRENT_HANDOFF.md` とGit差分を基準に復旧する。

## 12. Gitチェックポイント

- 意味のある作業単位またはエージェント交代前に、チェックポイントcommitを自由に作成できる
- WIPコミット前に秘密情報・一時生成物・意図しない変更を確認する
- コミットできない場合は、未コミットであることと理由を `CURRENT_HANDOFF.md` に明記する
- pushはユーザーの明示的許可またはタスク内指示がある場合のみ。許可を別タスク・別セッションへ持ち越さない

例：

```bash
git add <対象ファイル>
git commit -m "WIP: handoff <task-id> from Claude Code to Codex"
```

## 13. 同時実行

- 同じworktreeでClaude CodeとCodexを同時編集させない
- 順番に使う場合は、前担当を停止してから交代する
- 並行作業が必要なら別ブランチまたはGit worktreeを使用する
- 同じファイルを並行編集しない

## 14. 危険操作

以下はユーザーの明示的許可が必要。

- 本番公開、一括公開、一括生成、一括更新、一括削除
- DBマイグレーション、本番データの広範な変更
- protected slugのアフィリエイトリンク・料金記述・URLパラメータの変更（明白な異物除去はユーザーの都度承認時のみ可）
- 認証・決済・権限・APIの破壊的変更
- `force-regenerate`（実行のたびにユーザーの明示的許可が必要。包括許可は不可）
- 大規模リファクタリング
- 外部依存追加
- Git履歴の破壊的変更
- push

提案時に、目的、影響範囲、リスク、代替案、rollback、検証方法を示す。

## 15. 完了報告フォーマット

```md
## 実行結果
成功 / 一部成功 / 未完了

## 実装内容

## 変更した主要ファイル

## 実行した検証

## 検証範囲・対象件数

## 未実行の検証

## 発見した追加問題

## 未解決事項

## ユーザー判断が必要な事項

## 次に推奨する作業

## 関連コミット
```

「確認済み」とだけ書かず、方法・範囲・未検証箇所を明示する。
