# MoveWorth — Codex Instructions

> Codexは第二実装担当・フォールバック実装者として、Claude Codeの作業を安全に引き継ぐ。
> レビュー専用ではなく実装可能だが、既存差分とMoveWorthの安全ルールを尊重すること。

## 作業開始時

次の順で確認すること。

1. `AGENTS.md`
2. `CLAUDE.md`（MoveWorth固有の安全ルール）
3. `docs/AI_WORKFLOW.md`
4. `.ai/DECISIONS.md`
5. `.ai/CURRENT_HANDOFF.md`
6. `docs/BACKLOG.md`（関連する場合）
7. `git status`
8. `git diff`
9. 必要に応じて `git log --oneline -10`

コード変更前に `.ai/CURRENT_HANDOFF.md` を更新すること。

## 引き継ぎルール

- Claude Codeや別Codexセッションの説明を無条件に信用せず、実コードとGit差分で確認する
- 未コミット変更を理由なく削除・巻き戻し・全面書き換えしない
- 前担当の実装意図が不明な場合は、差分、関連ファイル、検証結果を調べてから変更する
- 同じworktreeでClaude Codeが動作中の場合は編集を開始しない
- エージェント交代時は、現在地と検証結果を共通ファイルへ残す

## 権限・禁止事項

ユーザーの明示的許可なしに、以下を実行しない。

- 本番公開、一括公開、一括生成、一括更新、一括削除
- DBマイグレーション、本番データの広範な変更
- protected slugのアフィリエイトリンク・料金記述・URLパラメータの変更
- `force-regenerate`（**実行のたびにユーザーの明示的許可が必要。セッション内の包括許可は不可**）
- 認証、決済、権限、APIの破壊的変更
- Git履歴の破壊的操作
- push

保護対象記事の明白な異物除去は、対象箇所と差分を示し、ユーザーの都度承認を得た場合のみ実行できる。異物除去の許可を、アフィリエイトリンク・料金記述・URLパラメータの変更許可として扱わない。

## 作業記録

- 開始時・重要な区切り・方針変更・危険操作前・検証後に `.ai/CURRENT_HANDOFF.md` を更新
- 終了時に `.ai/RECENT_ACTIVITY.md` へ要約を追記
- 恒久判断は `.ai/DECISIONS.md`
- 未完了タスクは `docs/BACKLOG.md`
- コード全文、diff全文、長いログは記録しない

## 完了報告

次を簡潔に報告すること。

1. 実行結果（成功 / 一部成功 / 未完了）
2. 実装内容
3. 変更した主要ファイル
4. 実行した検証と対象件数
5. 未実行の検証
6. 発見した追加問題
7. 未解決事項
8. ユーザー判断が必要な事項
9. 関連コミット
